# Asynchronous trong Node.js

==================================================

# Đồng bộ (Synchronous) là gì?

Trong mã đồng bộ,
các dòng code sẽ chạy lần lượt từng dòng một.

Ví dụ:

1. Require module
2. Đọc file
3. In dữ liệu ra console

Mỗi dòng phải chờ dòng trước hoàn thành.

Ví dụ:

const data = fs.readFileSync('test.txt', 'utf8');
console.log(data);

==================================================

# Blocking Code là gì?

Code đồng bộ còn được gọi là:

Blocking Code (mã chặn)

Vì:

- Trong lúc một tác vụ đang chạy
- Các dòng code khác phải chờ

Ví dụ:

- Đọc file lớn mất 5 giây
- Toàn bộ chương trình phải đứng chờ 5 giây

==================================================

# Vấn đề của Blocking Code trong Node.js

Node.js chỉ có:

1 Single Thread (1 luồng duy nhất)

Điều đó nghĩa là:

- Tất cả user dùng chung 1 luồng
- Nếu 1 user block thread
- Các user khác cũng phải chờ

Ví dụ:

- User A đọc file lớn
- User B login
- User C request data

=> B và C phải đợi A hoàn thành

Điều này rất tệ khi có:

- 1000 user
- 1 triệu user
- traffic lớn

==================================================

# Non-Blocking / Asynchronous là gì?

Để giải quyết vấn đề đó,
Node.js sử dụng:

Asynchronous Code (bất đồng bộ)

hay còn gọi là:

Non-Blocking Code

==================================================

# Ý tưởng của Asynchronous

Thay vì bắt chương trình chờ:

- Node.js sẽ đẩy tác vụ nặng ra background
- Sau đó tiếp tục chạy code khác
- Khi hoàn thành thì callback sẽ được gọi

==================================================

# Ví dụ bất đồng bộ

fs.readFile('test.txt', 'utf8', (err, data) => {
console.log(data);
});

console.log('Reading file...');

==================================================

# Thứ tự chạy

1. Node bắt đầu đọc file ở background
2. Không chờ kết quả
3. Chạy tiếp console.log()
4. Khi đọc file xong:
   callback được gọi
5. In dữ liệu ra console

==================================================

# Callback là gì?

Callback là:

- Một function truyền vào function khác
- Được gọi sau khi tác vụ hoàn thành

Ví dụ:

fs.readFile('test.txt', 'utf8', (err, data) => {
console.log(data);
});

Function:

(err, data) => {
console.log(data);
}

chính là callback.

==================================================

# I/O là gì?

I/O = Input / Output

Bao gồm:

- Đọc file
- Ghi file
- Request network
- Database
- API

Node.js được thiết kế tối ưu cho:

- Non-blocking I/O

==================================================

# Tại sao Node.js dùng callback nhiều?

Vì:

- Node.js cần tránh blocking
- Callback giúp xử lý kết quả sau này
- Trong lúc chờ:
  chương trình vẫn tiếp tục hoạt động

==================================================

# Callback KHÔNG tự động là Async

Quan trọng:

Có callback
KHÔNG có nghĩa là async.

Chỉ một số hàm của Node.js:

- fs.readFile()
- http request
- database query

mới thật sự async.

==================================================

# Callback Hell là gì?

Khi callback lồng callback quá nhiều:

readFile(() => {
readFile(() => {
readFile(() => {
writeFile(() => {

            });
        });
    });

});

Code sẽ:

- Rất khó đọc
- Khó debug
- Khó maintain

Đây gọi là:

Callback Hell

==================================================

# Giải pháp cho Callback Hell

JavaScript hiện đại dùng:

1. Promise
2. Async / Await

để viết async code sạch hơn.

Ví dụ:

async function readData() {
const data = await fs.promises.readFile('test.txt', 'utf8');
console.log(data);
}

==================================================

# Tóm tắt

## Synchronous

- Chạy từng dòng
- Blocking
- Chậm khi có tác vụ nặng

---

## Asynchronous

- Chạy nền
- Non-blocking
- Hiệu năng tốt hơn

---

## Node.js

- Single Thread
- Tất cả user dùng chung thread
- Phải tránh blocking code

---

## Callback

- Hàm chạy sau khi task hoàn thành
- Được dùng nhiều trong Node.js

---

## Callback Hell

- Callback lồng quá sâu
- Khó đọc
- Giải quyết bằng Promise hoặc Async/Await
